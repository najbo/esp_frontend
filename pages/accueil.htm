Gestion de l'enseignement spécialisé (ESP)

Le programme ESP est un outil informatique qui permet la gestion de l'enseignement spécialisé dans les écoles des degrés primaires et secondaires en Suisse. Il est basé sur le gestionnaire de contenu (CMS) OctoberCMS. En plus d'une interface d'utilisation facile et conviviale, le programme ESP permet la gestion de l'enseignement spécialisé par plusieurs utilisateurs et de manière décentralisée. En toute décontaction.

Les modules suivants sont proposés :

Menu Protocoles :

Le menu protocoles regroupe l'ensemble des fonctionnalités de base du programme ESP. Il s'agit en quelque sorte du menu de travail de l'outil ESP.

Module Description
Suivi des élèves Le module de base de l'ESP. Il regroupe le suivi des cas de l'enseigmenet spécialisé. Chaque suivi peut avoir plusieurs activités liées.
Activités Les activités sont des éléments du suivi. Ils détaillent les mesures ou des actions qui sont effectuées pour un suivi donné.
Il est également possible d'y rattacher des tiers (thérapeutes, enseignants, proches).
Pathologies Un suivi peut avoir une à plusieurs pathologies. Les pathologies peuvent aussi être affectées aux élèves.
Statuts des suivis Pour une meilleure vue d'ensemble, les suivis ont des étiquettes de statuts (en cours, en pause, terminé, etc...)
Types d'activités Chaque activité peut se voir attribuer une étiquette de type d'activité (constat, bilan, entretien, etc...)
Evolutions Les mesures peuvent avoir des évolutions (ICD / ESL/ ESG, etc..)
Statistiques Affiche quelques statistiques de base de l'outil ESP.

Menu Ressources humaines :

Le menu ressources humaines contient l'ensemble des modules liés aux différents personnes qui interviennent sur les dossiers de l'enseignement spécialisé.

Module Description
Elèves L'ensemble des élèves y sont regroupés. Il est possible d'associer les élèves à une école.
Interlocuteurs externes Les interlocuteurs externes sont les thérapeutes.
Enseignants L'ensemble des enseignants peut être saisi ici. Chaque enseignant est rattaché à une école.
Fonctions professionnelles Les fonctions professionnelles des interlocuteurs externes sont notés dans cette liste.
Rôles des proches Les proches des élèves peuvent avoir des rôles prédéfinis.

Menu Structures :

Le menu structures contient les données liées à la structures et à l'organisation de l'enseignement spécialisé.

Module Description
Ecole Chaque école peut être rattachée à un cercle scolaire. Il est aussi possible d'y ajouter les directeurs.
Cercles scolaires Les cercles scolaires contiennent les informations de base, telles que le responsable par défaut.
Année scolaires Les années scolaires peuvent être rattachées à chaque élève. Il est possible de procéder à un bouclement automatique beta
Programmes Les programmes scolaires contiennent les cycles et le type d'école.
On peut aussi indiquer le programme suivant à prendre en compte lors du bouclement automatique beta
Volées Chaque éleve est rattaché à une volée.
Types d'écoles Les types d'écoles sont mentionnés ici (école primaire, secondaire, etc...)
Passages Les différents passages d'une année à l'autre sont mentionnés ici. Utilisé lors du bouclement automatique des années beta

Menu Bases :

Le menu bases contient les listes et données de base de l'outil ESP. Ces informations sont en principe moins souvent accédées.

Module Description
Pays Les pays sont utilisés pour renseigner le pays d'origine d'un élève.
Langues Les langues sont utilisées pour renseigner la langue de base d'un élève.
Type de permis Types de permis de séjours ou d'établissement des élèves.
Genres Les genres sont le sexe de l'élève (homme, femme, autre)
Types de faits Pour chaque élève et enseignant, il est possible d'ajouter un événement/fait associé à un type spécifique (vacances, accident, début des rapports,...)
